# Proper Park Inha - 프로젝트 개발 규칙

본 문서는 Proper Park Inha 프로젝트의 개발 원칙과 코딩 규칙을 정의합니다.

## 프로젝트 개요

**프로젝트명**: Proper Park Inha (도시공원 품질 평가 연구)

**연구 목적**:
인천광역시 미추홀구의 도시공원 품질을 객관적으로 평가하고, 공원 접근성 사각지대를 도출하여 도시계획 개선방안을 제시

**연구 배경**:
- 도시공원은 도시민의 삶의 질에 직접적인 영향을 미치는 중요한 도시기반시설
- 기존 공원 평가는 주관적 설문조사나 단순 통계에 의존
- VLM(Vision Language Model) 기술을 활용한 객관적이고 정량적인 평가 필요
- 실제 공원 이용자 관점에서의 환경 품질 평가 부재

**연구 대상**:
인천광역시 미추홀구 도시공원 64개

**개발 환경**:
- Python 3.x
- Playwright (브라우저 자동화)
- Kakao Map API (로드뷰)
- Gemini (VLM 평가)
- QGIS (GIS 분석)

### 2.3 VLM 기반 공원 환경 평가 (Gemini)
**평가 Framework** (1-5점 척도):

1. **시설 관리 상태**
   - 평가 항목: 녹, 페인트 상태, 파손 여부
   - 중요도: 공원 이용 안전성과 직결

2. **휴게시설 충족도**
   - 평가 항목: 벤치, 그늘막, 테이블 등의 개수와 배치
   - 중요도: 이용자 편의성

3. **녹지 구성 다양성**
   - 평가 항목: 나무, 관목, 잔디 등 식재 요소의 다양성
   - 중요도: 생태적 가치 및 심미성

4. **개방성**
   - 평가 항목: 개방도 수준 (로드뷰 기반)
   - 중요도: 공원 접근성 및 안전감

5. **심미성**
   - 평가 항목: 녹지공간의 시각적 아름다움, 색채, 자연스러움
   - 중요도: 이용자 만족도

### 12. 최소한의 에러 핸들링 (Minimal Error Handling)
**원칙**: 요청하지 않은 에러 처리 코드 작성 금지
- 예외 상황 발생 시 → 에러 반환만
- try-except 남용 금지
- 에러 처리 로직은 명시적 요청 시에만 구현

**기본 원칙**:
```python
# ✅ 기본 방식 (에러 처리 요청 없을 때)
def divide(a, b):
    """나눗셈 수행"""
    return a / b  # ZeroDivisionError는 호출자가 처리

# ✅ 에러 처리 요청 시에만
def divide_safe(a, b):
    """안전한 나눗셈 (에러 처리 포함)"""
    if b == 0:
        raise ValueError("0으로 나눌 수 없습니다")
    return a / b
```
